## Submission Rules

- Read each target README, contributing guide, license, issue template, PR
  template, and sorting rules before submitting.
- Search the target repo, open PRs, open issues, closed issues, and accepted
  entries for `hermes-tweet`, `xquik`, `tweetclaw`, and `x-twitter-scraper`.
- Prefer a small, accurate PR over an issue when the target accepts PRs.
- Do not submit duplicate, rejected, credential-gated, paid-only, abandoned, or
  policy-incompatible entries.
- Use concise descriptions that explain developer value without keyword
  stuffing or unsupported claims.
- Record durable no-op reasons in this file only when they change future action.
